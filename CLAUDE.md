# このリポジトリで作業する前に必ず読むこと

## 最初の一手（例外なし）

このリポジトリで何かを読む・編集する・調査する前に、**一番最初のツール呼び出しとして** 次を実行する。

```
python scripts/check_live_sync.py
```

- **exit code 0（一致）** → 通常通り作業を進めてよい。
- **exit code 非0（不一致、または wrangler の pages 権限が復活している警告）** → その場で作業を止め、差分の内容をそのままユーザーに報告する。差分の原因を確認・解決するまで、記事の編集・pages.json等の上書き・再デプロイを行わない。

## なぜこれが必須なのか

2026-07-15、この確認をせずに「`git status`がoriginと同期している」ことだけを根拠に作業を始めたセッションが、本番にしか存在しなかった108記事に気づかないまま上書きしかけた事故があった。`git status`の同期表示は「ローカルとGitHub」の一致は保証するが、「GitHubとCloudflare本番」の一致は保証しない（本番にはGit を経由しない書き込み経路が過去に複数回あったため）。

以後、恒久的な仕組み（このマシンのwranglerからPages書き込み権限を除去・Cloudflareビルド時の本番差分チェック等）を導入したため事故の再発確率は下がったが、**それでも「本番だけが先に進んでいる」可能性をゼロと断定してはいけない**。この最初の一手は、その断定を避けるための最終防衛線であり、省略してはならない。

詳細な経緯は `README.md`「本番の巻き戻り・記事消失を防ぐ安全装置」の節を参照。
